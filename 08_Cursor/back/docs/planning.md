# 학생용 정보공유 커뮤니티 기획서 (MVP)

## 1. 서비스 개요

**서비스 한 줄 정의**

> 학생들이 공부하며 겪은 과정을 기록하고, 막힌 지점을 공유하는 학습 커뮤니티

**기획 의도**
- 완성도 높은 글보다 **학습 과정과 경험 공유**에 초점
- 질문과 기록이 자연스럽게 쌓이는 구조
- 학생들이 부담 없이 글을 쓰고 소통할 수 있는 환경 제공

---

## 2. 타겟 사용자

- 대상: **학생**
  - 전공/비전공 무관
  - 학습 과정 중 기록과 질문이 필요한 사용자
- 실명 기반은 아니지만 **익명은 허용하지 않음**
  - 닉네임 필수
  - 책임감 있는 커뮤니티 분위기 유지

---

## 3. 커뮤니티 방향성

### 지향점
- 기록 = 가치
- 질문 = 환영
- 미완성 글 OK
- 짧은 글 OK

### 지양점
- 과도한 전문성 경쟁
- 정답만 요구하는 분위기
- 글 작성에 대한 심리적 부담

---

## 4. 정보 구조 (IA)

### 카테고리 구조 (행동 기반, 고정)

| 카테고리 | 설명 |
|---|---|
| 📝 오늘 배운 것 | 오늘 학습한 내용 기록 |
| ❓ 막힌 것 | 오류, 개념 질문, 해결 요청 |
| 💡 짧은 팁 | 간단한 팁, 노하우 공유 |
| 📌 정리 노트 | 개념 정리, 요약 글 |

- 카테고리는 고정
- 태그 기능 없음 (초기 복잡도 제거)

---

## 5. 핵심 기능 정의 (MVP)

### 필수 기능

| 기능 | 설명 |
|---|---|
| 회원가입 / 로그인 | 이메일 + 비밀번호 |
| 글 작성 | 카테고리 선택 + 제목 + 본문 |
| 글 조회 | 목록 / 상세 |
| 댓글 | 단일 댓글 구조 |
| 좋아요 | 게시글 반응 |

---

## 6. 화면 구성

### 6.1 메인 페이지

구성 요소:
1. 서비스 소개 문구  
   - “오늘 배운 것, 막힌 것, 전부 기록하세요”
2. 카테고리 필터 버튼
3. 게시글 목록 (Feed)

게시글 목록 표시 정보:
- 제목
- 작성자 닉네임
- 카테고리
- 댓글 수 / 좋아요 수
- 작성 시간

정렬:
- 최신순 고정

---

### 6.2 게시글 상세 페이지

구성 요소:
- 제목
- 작성자
- 카테고리
- 본문 (Markdown 지원 가능)
- 좋아요 버튼
- 댓글 목록
- 댓글 작성 입력창

UX 포인트:
- 댓글 입력창 항상 노출
- 댓글 유도 문구 제공  
  > “경험을 공유해 주세요”

---

### 6.3 글 작성 페이지

입력 항목:
- 카테고리 (필수)
- 제목
- 본문

특징:
- 글 템플릿 없음
- 파일 업로드 없음 (1차 제외)
- Placeholder 문구로 방향성만 제시

예:
- “오늘 어떤 걸 배웠나요?”
- “어디서 막혔나요?”

---

### 6.4 로그인 / 회원가입

- 이메일
- 비밀번호
- 닉네임 (중복 불가)

---

### 6.5 마이페이지

구성:
- 내가 작성한 글 목록
- 받은 좋아요 총합
- 작성한 댓글 수

목적:
- 학습 기록의 누적 확인
- 사용자 동기 부여

---

## 7. 기능 단위 정리 (백엔드 기준)

### User
- 회원가입
- 로그인
- 내 정보 조회

### Post
- 게시글 작성
- 게시글 목록 조회
- 게시글 상세 조회
- 게시글 수정 / 삭제 (작성자만)

### Comment
- 댓글 작성
- 댓글 조회

### Like
- 좋아요 등록 / 취소 (토글)

---

## 8. 데이터 모델 (ERD 초안)

### User
- id
- email
- password
- nickname
- role
- created_at

### Post
- id
- user_id (FK)
- category (ENUM)
- title
- content
- like_count
- created_at
- updated_at

### Comment
- id
- post_id (FK)
- user_id (FK)
- content
- created_at

### Like
- id
- post_id (FK)
- user_id (FK)
- created_at

> `like_count`는 조회 성능을 위한 캐시 필드로 사용

---

## 9. 기술 스택 가이드 (예시)

- Backend: Spring Boot
- Auth: Session 또는 JWT (단순 구성)
- DB: MySQL
- Frontend: React
- Deployment: EC2 + RDS (추후)
